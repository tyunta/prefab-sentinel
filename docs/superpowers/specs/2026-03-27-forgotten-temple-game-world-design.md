# Forgotten Temple — VRChat 2人協力ファンタジー探索ゲームワールド

**Date**: 2026-03-27
**Status**: Draft
**Project**: D:\VRChatProject\PS-WORLD-TEST

## 概要

2人で古代の神殿を探索するVRChat協力ゲームワールド。部屋ごとに異なるギミック（たいまつ、圧力板、鏡パズル）を2人で協力して解き、最奥の祭壇で秘宝を発見してクリアする。

### 要件サマリ

| 項目 | 内容 |
|---|---|
| プレイ人数 | 2人 |
| ジャンル | 協力探索 |
| 世界観 | ファンタジー（古代神殿） |
| プレイ時間 | 15〜30分 |
| 協力スタイル | 常に一緒に行動 |
| リプレイ性 | 1回クリアで完結（一本道） |

## マップ構成

ロビー → 4つの部屋 → 祭壇の直線配置。各部屋の間は短い通路で繋ぎ、扉をクリア条件でゲート制御する。

| エリア | 役割 | 目安時間 |
|---|---|---|
| ロビー | チュートリアル。操作説明と簡単な協力アクション（2人で扉を開ける） | 2〜3分 |
| 部屋1: 灯火の間 | たいまつギミック。1人が照らし、もう1人がヒントを伝える | 4〜5分 |
| 部屋2: 重圧の間 | 圧力板ギミック。2人同時に乗るとクリア | 3〜4分 |
| 部屋3: 鏡の間 | 光反射パズル。鏡2枚を2人で操作して光をクリスタルに当てる | 5〜7分 |
| 部屋4: 試練の間 | 複合ギミック。灯火＋圧力板＋鏡の総合パズル | 5〜7分 |
| 祭壇 | クリア演出。秘宝発見、パーティクル、記念撮影スポット | 2〜3分 |

## ギミック詳細

### ロビー — チュートリアル

- 2人がそれぞれの台座に立つと、中央の大扉がゆっくり開く
- 壁の碑文に「2人で協力して進め」と表示
- 操作に慣れる時間を兼ねる

### 部屋1: 灯火の間

- 入口にたいまつ（VRC_Pickup）が1本
- 部屋は暗く、たいまつ持ちの周囲だけ Point Light で照らされる
- 壁に3つの通路があり、正解の通路だけシンボルが刻まれている
- たいまつを持っていない側の壁にヒントの碑文（「太陽の印を追え」）
- **協力ポイント**: たいまつ持ちが照らし、もう1人がヒントを伝える口頭コミュニケーション

### 部屋2: 重圧の間

- 部屋の両端に光る圧力板が1つずつ（距離を離す）
- 2人が同時（3秒以内の猶予）に乗ると扉が開く
- 片方だけだとカウントダウン後にリセット
- **協力ポイント**: 声で「せーの」のタイミング合わせ

### 部屋3: 鏡の間

- 天井の穴から光のビーム（LineRenderer）が1本
- 回転可能な鏡が2枚（Interact で90度ずつ回転）
- 光を 鏡A → 鏡B → 扉上のクリスタル に当てると扉が開く
- **協力ポイント**: 「もうちょい右」「そこ！」の空間的コミュニケーション

### 部屋4: 試練の間

- 暗い部屋（灯火の間の要素）+ 圧力板2つ + 鏡1枚
- 手順:
  1. たいまつで壁のヒントを見る
  2. 鏡の正しい角度を判断
  3. 1人が鏡を合わせ、もう1人が圧力板に乗ると光源が起動
  4. 光が鏡を通ってクリスタルに当たる
- **協力ポイント**: 全ギミックの総合。手順の相談と役割分担

### 祭壇

- 秘宝（光り輝くオブジェクト）が台座に浮いている
- 2人で同時に Interact すると秘宝が解放
- パーティクル演出 + BGM変化
- 記念撮影用の台座（2人並べるスポット）とクリアタイムの表示

## 技術設計

### 同期方式

- Manual sync を基本とする（ギミック状態の同期頻度が低いため）
- 各部屋に1つの管理スクリプト（UdonSharpBehaviour）を置き、部屋の状態を `[UdonSynced]` で同期
- オーナーシップ: 部屋に最初に入ったプレイヤーがオーナー。Pickup 系は VRCSDK の自動オーナー移譲に任せる

### スクリプト構成

| スクリプト | 責務 |
|---|---|
| `GameManager` | 全体進行管理。現在の部屋番号、クリア状態、タイマー |
| `DoorController` | 扉の開閉。条件（bool）を満たしたらアニメーション再生 |
| `PressurePlate` | 圧力板。OnPlayerTriggerEnter/Exit で検知、2枚の同時検知を判定 |
| `TorchLight` | たいまつ追従ライト。Pickup の OnPickup/OnDrop でライト ON/OFF |
| `MirrorRotator` | 鏡の回転。Interact で90度回転、現在角度を同期 |
| `LightBeam` | LineRenderer で光線描画。鏡の角度からレイキャストして反射計算 |
| `AltarFinish` | 祭壇のクリア演出。2人 Interact 検知 → パーティクル + BGM |

### 同期の流れ（例: 圧力板）

1. ローカルで `OnPlayerTriggerEnter` 検知
2. `SendCustomNetworkEvent(Target.All, "OnPlateActivated")` で両者に通知
3. `GameManager` が2枚とも active なら `DoorController.Open()` を呼ぶ

### パフォーマンス目標

| 項目 | 目標 |
|---|---|
| Quest 対応 | Point Light 同時2個まで、リアルタイムシャドウはたいまつのみ |
| ポリゴン数 | 全体 50k tri 以下（ローポリ石造りスタイル） |
| マテリアル数 | 共通テクスチャアトラスで 5〜8 以内 |
| ライティング | ベイクドライティング基本、たいまつのみリアルタイム |
| FPS | Quest で 30 以上維持 |

### レイヤー・コリジョン

- Default: 静的メッシュ
- PlayerLocal / Player: VRC 標準
- Pickup: たいまつ
- MirrorReflection: 使用しない
- 圧力板: OnPlayerTriggerEnter で十分、専用レイヤー不要

## エラーハンドリング

| ケース | 対処 |
|---|---|
| 片方が途中離脱 | `OnPlayerLeft` で検知 → 待機画面表示。ギミック状態はリセットしない |
| たいまつを落とした場所に戻れない | 30秒操作がなければ元の台座にリスポーン |
| 圧力板のタイミングが合わない | 猶予3秒、リトライ回数制限なし |
| プレイヤー落下 | 各部屋にリスポーンポイント配置。落下判定コライダーを床下に設置 |
| 光線パズルで詰まる | 鏡の選択肢を4方向（90度刻み）に限定、総当たりでも解ける設計 |

## 演出

- **BGM**: 探索中は静かなアンビエント。部屋クリア時に短いファンファーレ。祭壇で壮大な曲に切り替え
- **SE**: 扉の開閉音、圧力板の作動音、たいまつの燃える音、光線命中時のチャイム
- **ライティング**: 基本薄暗い。クリア時に部屋が明るくなる演出で達成感を強調
- **パーティクル**: 祭壇クリア時に光の粒子。鏡パズル正解時にクリスタルが輝く
